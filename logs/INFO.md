# logs/

This folder stores runtime logs generated by each module.

Typical log files:
- `pc.log`
- `gli.log`
- `indexador.log`
- `buscador.log`

What is recorded:
- Module start and finish events.
- Processing phase boundaries.
- Input loading and item counts.
- Average processing times.
- Execution errors, when present.

Notes:
- Log files are regenerated/updated on each execution.
